# zkverify-rs

A high-performance, command-line Groth16 zero-knowledge proof verifier for the BN254 elliptic curve. This tool enables cryptographic verification of proofs generated by snarkjs, arkworks, and other Groth16-compatible proof systems. It is designed for security auditing, proof validation pipelines, and privacy-preserving applications that require provably correct computation without revealing sensitive inputs.

## Installation

### Prerequisites

- **Rust 1.56+**: Install from [https://rustup.rs/](https://rustup.rs/)
- **Cargo**: Included with Rust

### Build from Source

```bash
git clone <repository>
cd zkverify-rs
cargo build --release
```

The compiled binary will be at `target/release/zkverify-rs`.

## Usage

### Basic Verification

```bash
./target/release/zkverify-rs \
  --vkey examples/vkey.json \
  --proof examples/proof.json \
  --inputs examples/public.json
```

### Generate Test Proofs

To create example proofs for testing:

```bash
cargo run --bin generate_test_proof
```

This generates three files in `examples/`:
- `proof.json` - The Groth16 proof
- `vkey.json` - The verification key
- `public.json` - Public inputs

### As a Library

Add to your `Cargo.toml`:

```toml
[dependencies]
zkverify-rs = { path = "./zkverify-rs" }
```

## File Format

### Proof Format (snarkjs-compatible)

```json
{
  "pi_a": ["x", "y", "1"],
  "pi_b": [["x0", "x1"], ["y0", "y1"], ["1", "0"]],
  "pi_c": ["x", "y", "1"],
  "protocol": "groth16",
  "curve": "bn254"
}
```

### Verifying Key Format

```json
{
  "protocol": "groth16",
  "curve": "bn254",
  "nPublic": 1,
  "vk_alpha_1": ["x", "y", "1"],
  "vk_beta_2": [["x0", "x1"], ["y0", "y1"], ["1", "0"]],
  "vk_gamma_2": [["x0", "x1"], ["y0", "y1"], ["1", "0"]],
  "vk_delta_2": [["x0", "x1"], ["y0", "y1"], ["1", "0"]],
  "ic": [["x", "y", "1"], ...]
}
```

### Public Inputs Format

```json
["input0", "input1", ...]
```

Each input is a decimal string representing a field element in BN254.

## Technical Details

### Groth16 Verification Equation

Groth16 is a zero-knowledge proof system that works by verifying the equation:

$$e(A, B) = e(\alpha, \beta) \cdot e\left(\sum_{i=0}^{n} l_i \cdot IC[i], \gamma\right) \cdot e(C, \delta)$$

Where:
- $e$ is the Tate pairing on the BN254 curve
- $A, B, C$ are proof elements
- $\alpha, \beta, \gamma, \delta$ are trusted setup parameters
- $l_i$ are the public input values
- $IC[i]$ are precomputed linear combination coefficients
- $n$ is the number of public inputs

### Why This Matters for ZK Security

1. **Soundness**: If the prover doesn't know a valid witness, they cannot forge a proof that passes this equation with negligible probability.

2. **Privacy**: The proof reveals no information about private inputs; only public inputs affect verification.

3. **Non-Interactive**: Unlike interactive protocols, Groth16 requires only a single message from prover to verifier, enabling asynchronous verification.

4. **Trusted Setup**: Security relies on a structured reference string (SRS) generated in a ceremony. Compromise of SRS parameters breaks the scheme.

5. **Efficiency**: Groth16 has constant proof size (3 group elements) and verification time, making it production-ready for blockchain and privacy systems.

## Architecture

- **`src/parser.rs`** - JSON parsing and snarkjs format conversion
- **`src/verifier.rs`** - Core Groth16 verification logic
- **`src/main.rs`** - CLI interface using `clap`
- **`src/bin/generate_test_proof.rs`** - Test proof generation

## Dependencies

- `ark-groth16` - Groth16 proof system
- `ark-bn254` - BN254 curve implementation
- `ark-ec`, `ark-ff`, `ark-std` - Cryptographic primitives
- `ark-relations` - Constraint system for circuit building
- `serde`, `serde_json` - JSON serialization
- `clap` - CLI argument parsing
- `rand` - Random number generation

## Examples

### Verify a Proof

```bash
$ cargo run --release -- \
  --vkey examples/vkey.json \
  --proof examples/proof.json \
  --inputs examples/public.json

✓ Proof verified successfully
```

### Batch Verification

Create a script to verify multiple proofs:

```bash
#!/bin/bash
for dir in proofs/*/; do
  echo "Verifying $(basename $dir)..."
  ./target/release/zkverify-rs \
    --vkey "$dir/vkey.json" \
    --proof "$dir/proof.json" \
    --inputs "$dir/public.json" || exit 1
done
```

## Performance

- Verification time: ~50-100ms per proof (depends on number of public inputs)
- Memory usage: ~50MB for typical verification
- Compile time: ~2-3 minutes (release build)

## Security Considerations

1. **Trusted Setup**: Verify the trusted setup ceremony for the proof system
2. **Implementation Risk**: Use audited versions of arkworks
3. **Side-Channel**: Run on secure hardware when verifying sensitive proofs
4. **Key Management**: Protect verifying keys; they should be publicly available but immutable

## Troubleshooting

### "Failed to parse Fr from..."
- Check that public inputs are valid decimal integers
- Ensure they fit within the BN254 scalar field (approximately $2^{254}$)

### "Failed to parse proof JSON"
- Verify the proof format matches snarkjs structure
- Check that coordinate arrays have correct dimensions

### "Proof verification failed"
- Confirm proof was generated with the same setup and vkey
- Ensure public inputs match what the prover used
- Check that the circuit specification hasn't changed
